# Problem Statement

Imagine that your mobile checkout drops at the last step when internet quality is unstable.

## Symptoms

- Users lose entered payment details.
- Retry path is unclear.
- Drop-off spikes on lower-end devices.

## Goal

Reduce failed checkout exits by introducing robust local recovery and clear retry messaging.

## Proposed Approach

1. **Persist form state to sessionStorage** on every field blur. If the page reloads, restore from storage.
2. **Queue the payment request** using a service worker. If the network drops mid-submit, retry automatically when connectivity returns.
3. **Show a clear status bar** — "Saving...", "Retrying...", "Payment confirmed" — so users never wonder what's happening.

## Edge Cases

- What if the payment actually went through but the confirmation response was lost? Need an idempotency key.
- What about users who clear their browser storage between attempts? We lose the recovery path. Acceptable trade-off.
