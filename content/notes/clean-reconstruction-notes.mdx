# Clean Reconstruction

Here is the clean reconstruction in text form, with examples and practical caveats added.

## Key Principles

- Normalize all examples before comparing outputs.
- Keep intermediate steps explicit for debugging.
- Prefer small test fixtures with known edge cases.

## Example Workflow

1. Take the raw input and strip whitespace variations.
2. Run through the transformer pipeline.
3. Compare output against the golden file using `diff --unified`.

```bash
# Quick comparison
diff --unified expected.txt actual.txt | head -20
```

## Caveats

- Floating point comparisons need epsilon tolerance. Don't use strict equality.
- Unicode normalization matters — NFC vs NFD can cause silent mismatches.
- When in doubt, log the intermediate representation. It's always a pipeline bug.
